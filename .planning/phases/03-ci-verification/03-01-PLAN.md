Create `scripts/test-worker.mjs` -- a Node.js smoke test script (~80 lines) that validates the Worker starts and handles requests. Follow existing project conventions: `#!/usr/bin/env node`, `.mjs` extension, no external test framework.

**Script structure:**

1. **Constants:**
   - `PORT = Number(process.env.WORKER_TEST_PORT) || 18787` (env var override for CI, non-default port to avoid conflicts)
   - `TIMEOUT_MS = 30_000` (30 seconds max for wrangler to become ready)
   - `MASTER_KEY = "smoke-test-key"`
   - `CONFIG = JSON.stringify({ version: 2, masterKey: MASTER_KEY, providers: [] })`

2. **Spawn wrangler dev:**
   ```javascript
   import { spawn } from "node:child_process";
   const child = spawn("npx", [
     "wrangler", "dev",
     "--port", String(PORT),
     "--show-interactive-dev-session", "false",
     "--var", `LLM_ROUTER_CONFIG_JSON:${CONFIG}`
   ], { stdio: ["ignore", "pipe", "pipe"] });
   ```

3. **Wait for ready signal:** Listen to BOTH stdout AND stderr (wrangler may route output to either stream across versions). Match on the string `Ready on` as the trigger. Set a `setTimeout` for `TIMEOUT_MS` that rejects with a timeout error if the ready signal never arrives.

4. **Send health check:** After ready signal detected:
   ```javascript
   const res = await fetch(`http://localhost:${PORT}/health`, {
     headers: { "Authorization": `Bearer ${MASTER_KEY}` }
   });
   ```

5. **Assert response:** Check `res.status === 200`. Parse body as JSON and verify it contains `status: "ok"`. Print result to stdout. If assertion fails, print the actual status and body, then `process.exit(1)`.

6. **Cleanup (try/finally):** Wrap the entire spawn-wait-check sequence in try/finally. The finally block MUST call `child.kill("SIGTERM")`. Also register signal handlers for SIGINT and SIGTERM that kill the child process (follow the pattern from `scripts/dev.mjs`). Set `child.on("exit", ...)` to track whether it already exited to avoid killing a dead process.

7. **Exit:** `process.exit(0)` on success, `process.exit(1)` on any failure.

**Console output format:**
```
Worker smoke test starting on port 18787...
Waiting for wrangler dev ready signal...
Worker ready. Sending health check...
Health check passed: 200 {"status":"ok","timestamp":"...","providers":0}
Worker smoke test PASSED
```

On failure:
```
Worker smoke test FAILED: Health check returned 503 {"error":"..."}
```
or:
```
Worker smoke test FAILED: Timed out after 30000ms waiting for wrangler dev to start
```

**Then update `package.json`:** Add `"test:worker": "node ./scripts/test-worker.mjs"` to the `scripts` section. Place it after the `test:amp-smoke` entry (alphabetical grouping of test: scripts). Do NOT modify any other scripts.

**Things to NOT do:**
- Do NOT use `--local` flag (local mode is default in wrangler v3+, flag is unnecessary)
- Do NOT use a bash shell script (project convention is .mjs)
- Do NOT use `setTimeout` with `sleep()` instead of stream-based ready detection (race condition)
- Do NOT forget auth header (Worker returns 401 without it)
- Do NOT hardcode port 8787 (default wrangler port -- conflicts with running dev server)
- Do NOT leave orphaned wrangler processes on failure (try/finally + signal handlers mandatory)
- Do NOT import from project source files (this is a standalone process test)
