```toml
# Optional non-secret vars
[vars]
ENVIRONMENT = "production"

# Required secret(s):

#                          llm-router deploy --export-only=true --out=.llm-router.worker.json
#                          wrangler secret put LLM_ROUTER_CONFIG_JSON < .llm-router.worker.json
#
# Optional override:
# LLM_ROUTER_MASTER_KEY   - Overrides config.masterKey at runtime
```

Changes from current file:

2. Line 4: Added `compatibility_flags = ["nodejs_compat"]` (CONF-02). This enables the unenv polyfill layer for all node:* modules. All 6 problematic modules (node:http, node:crypto, node:child_process, node:fs, node:path, node:os) will resolve.
3. Lines 8-15: Added `find_additional_modules = true` and `[[rules]]` section (CONF-03). The `find_additional_modules` setting prevents esbuild from inlining dynamic imports. The `[[rules]]` entry with `type = "ESModule"` and `globs = ["runtime/subscription-*.js"]` marks subscription modules as external. CRITICAL: globs are relative to base_dir which defaults to the directory of main entrypoint (src/), NOT the project root. So "runtime/subscription-*.js" matches "src/runtime/subscription-*.js". Do NOT prefix with "src/".
4. Removed the generic "No custom routes" comment (lines 7-8 of original) since the file now has meaningful configuration in that area.
5. Preserved [vars] and secret comments unchanged.

Things to NOT do:

- Do NOT use `./` prefix in globs (use `runtime/subscription-*.js` not `./runtime/subscription-*.js`)
- Do NOT use `EsModule` casing (use `ESModule` -- capital E, capital S, capital M per v4 config reference)
- Do NOT add a `[build]` section (no build pipeline needed)
- Do NOT add `nodejs_compat_v2` as an explicit flag (redundant -- auto-enabled by date >= 2024-09-23)
